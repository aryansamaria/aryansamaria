TechStore AI Support

AI Customer Support · Claude Hooks · MCP · Skills

Production-grade AI customer support system for a fictional electronics retailer — Claude reads live PostgreSQL order data, handles escalations, and runs automatic hooks for logging, security, and archiving.

Problem

Traditional AI support required months of boilerplate: intent detection, tool dispatch layers, security middleware, conversation archiving, and slash command parsers — each a separate engineering effort with brittle glue code.

Approach

Built around Claude Code's three pillars. Hooks — shell scripts firing on Claude events — handle query logging, dangerous command blocking, audit trails, and transcript archiving automatically. Skills (Markdown files in .claude/commands/) become slash commands like /lookup-order and /escalate with business logic in plain English. MCP server exposes six PostgreSQL tools via @mcp.tool() decorated Python functions over stdin/stdout JSON-RPC — no API layer, no dispatch code.

System

Claude Code CLI + React Web App → FastAPI (SSE streaming agentic loop) → MCP Server (JSON-RPC child process) → PostgreSQL. Four hooks: UserPromptSubmit (query log), PreToolUse/Bash (security guard blocking rm -rf etc.), PostToolUse (audit trail), Stop (transcript archive). Six MCP tools: get_order, get_customer, get_orders_by_customer, search_kb (PostgreSQL FTS), create_escalation, save_ticket_summary.

Outcome

Full system in a single session — 4 automatic hook behaviours, 4 custom slash commands, 6 live database tools, and a 6-page React dashboard with streaming chat showing real-time tool call cards. Every session logged, audited, and archived with zero extra infrastructure.
